# Test Data

This directory contains captured AWS event payloads from the test incident scenario.

## Files

These files are generated by running `scripts/capture-alarm-event.sh`:

- **cloudwatch-alarm-event.json**: The EventBridge event triggered by CloudWatch Alarm
- **sample-metrics.json**: Actual CloudWatch metrics (CPU, network, disk)
- **sample-logs.json**: CloudWatch Logs entries (if CloudWatch Agent configured)
- **sample-cloudtrail-events.json**: Recent CloudTrail events for the EC2 instance

## Purpose

These files serve as:
1. **Test fixtures** for unit tests
2. **Documentation** of actual AWS event schemas
3. **Reference data** for understanding the system's inputs

## Generating Test Data

```bash
# 1. Deploy test infrastructure
cd terraform/test-scenario
terraform apply

# 2. Trigger the alarm
cd ../..
./scripts/trigger-test-alarm.sh

# 3. Capture event payloads
./scripts/capture-alarm-event.sh
```

## Using in Tests

```python
import json

# Load test fixture
with open('test-data/cloudwatch-alarm-event.json') as f:
    alarm_event = json.load(f)

# Use in unit test
def test_metrics_collector():
    result = metrics_collector.handler(alarm_event, {})
    assert result['status'] == 'success'
```

## Note

These files are gitignored by default since they contain AWS account-specific data. Each developer should generate their own test data.
